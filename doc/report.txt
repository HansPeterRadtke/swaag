SWAAG test and evaluation report
================================

Scope of this pass
------------------
This pass fixed the currently known runtime/import regressions, cleaned the
default non-benchmark test surface, and made the testing architecture explicit
as three lanes:

1. deterministic correctness
2. agent-loop regression
3. live agent evaluation

Real bugs fixed
---------------
1. Background/runtime lifecycle bugs are fixed under the focused failing cases:
   - background `run_tests` completion is detected and propagated
   - the runtime keeps working while background work runs
   - the runtime enters an explicit wait state when only background work remains

2. Benchmark import hygiene is fixed:
   - importing `swaag.benchmark` no longer eagerly requires optional official
     benchmark dependencies such as `datasets`
   - benchmark-specific dependencies are imported lazily only when those paths
     are actually used

3. Scripted benchmark verification for respond steps is fixed:
   - the benchmark verification fallback now returns criterion names correctly
   - respond-step verification no longer fails spuriously and trigger pointless
     replans in the generated benchmark catalog

4. The default source-tree write-safety policy now stays clean:
   - benchmark/report helpers use the central `swaag.fsops` helpers instead of
     direct `Path.write_text` / `mkdir` / `unlink` calls scattered through the
     source tree

Testing architecture now implemented
------------------------------------
The repository now distinguishes:

1. Deterministic correctness lane
   - imports, smoke tests, unit tests, harness checks, packaging/integration
   - expected to stay at 100 percent

2. Agent-loop regression lane
   - real runtime/orchestrator/tool-loop behavior
   - replayed or scripted model responses only
   - includes a test-only record/replay client keyed by the normalized full
     request payload plus request metadata

3. Live agent evaluation lane
   - real model in the loop
   - full agent task execution
   - five ordered difficulty tiers:
     - extremely_easy
     - easy
     - normal
     - hard
     - extremely_hard

4. Combined reporting
   - machine-readable JSON
   - human-readable markdown
   - per-lane percentages
   - live per-tier percentages
   - live per-task percentages and rubric breakdowns
   - one final overall percentage as the arithmetic mean of the three lane
     scores

Primary commands
----------------
Fast non-live combined view:

  python3 -m swaag.benchmark evaluate --clean --output /tmp/swaag-eval

Explicit agent-loop regression lane:

  python3 -m swaag.benchmark regression --all --clean --output /tmp/swaag-regression

Authoritative three-lane score:

  python3 -m swaag.benchmark three-lane-evaluate --clean --live-subset --output /tmp/swaag-three-lane
