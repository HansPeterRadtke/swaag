SWAAG test and evaluation report
================================

Scope of this repo state
------------------------
The repository distinguishes two user-facing test categories:

1. Deterministic correctness tests
2. Agent behavior tests
   - cached mode
   - no-cache validation mode

Deterministic correctness tests are the mechanical bug-detection surface.
Agent behavior tests exercise the full runtime, orchestration, tools,
verification, and reporting path.

Current command surface
-----------------------
Deterministic correctness tests:

  python3 -m pytest -q -x

Agent behavior tests (cached mode):

  python3 -m swaag.benchmark agent-tests --mode cached --clean --validation-subset --output /tmp/swaag-agent-tests-cached

Agent behavior tests (no-cache validation mode):

  python3 -m swaag.benchmark agent-tests --mode no-cache-validation --clean --validation-subset --output /tmp/swaag-agent-tests-validation

Authoritative combined score:

  python3 -m swaag.benchmark test-categories --clean --validation-subset --output /tmp/swaag-test-categories

Replay cache notes
------------------
The cached-mode path uses a test-only record/replay client keyed by:

- normalized request payload
- request metadata
  - model base URL
  - completion endpoint
  - model profile
  - structured output mode
  - seed
  - explicit task/mode metadata where the benchmark wrapper adds it

`timeout_seconds` is stored with the request envelope for observability but is
not part of the hash because it is a transport setting, not a semantic model
input.

Validation subset
-----------------
The no-cache validation subset keeps at least ten distinct tasks in each
ordered difficulty tier:

- extremely_easy
- easy
- normal
- hard
- extremely_hard

The category reports expose:

- per-task percentages
- per-task rubric breakdowns
- per-tier percentages for no-cache validation mode
- per-category percentages
- final overall percent
